# API Design

## Status

Standards defined. Full endpoint catalog to be created during implementation.

## Overview

All APIs follow REST conventions and are documented via OpenAPI (auto-generated by FastAPI).

## Key Decisions

- **REST**: Standard REST conventions for all endpoints
- **Versioning**: APIs are versioned via URL prefix (`/api/v1/`)
- **Consistent response envelopes**: All responses use a standard structure
- **Pagination**: All list endpoints support cursor or offset pagination

## Standards

### Response Envelope

```json
{
  "data": { ... },
  "meta": { "page": 1, "total": 100 },
  "errors": []
}
```

### Error Response

```json
{
  "data": null,
  "errors": [
    { "code": "VALIDATION_ERROR", "field": "price", "message": "Price must be greater than 0" }
  ]
}
```

### Monetary Values

- API responses return monetary values as formatted strings with currency code: `"AED 1,250,000"`
- API inputs accept monetary values as integers (smallest unit): `125000000` (representing AED 1,250,000 in fils)

### Naming Conventions

- Endpoint paths: `snake_case` (e.g., `/api/v1/payment_plans/`)
- JSON keys: `snake_case`
- UUIDs: returned as strings in standard UUID format
